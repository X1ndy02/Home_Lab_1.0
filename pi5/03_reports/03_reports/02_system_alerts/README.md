# System Alerts

These folders are for event-driven mail: security incidents, service failures, state changes, and hardware warnings.

Recommended layout:

- group by alert type
- keep one month folder at the next level
- store one timestamped file or one timestamped folder per sent mail

Example:

- `pi_monitor/2026-03/2026-03-26T23-06-18_ssh_login_warn.txt`
