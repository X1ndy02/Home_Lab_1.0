# Pi Monitor Alerts

Source: `/srv/monitoring/bin/monitor-checks.sh`

Cadence: event-driven from the minute-based `pi-monitor.timer`

This folder is for per-event monitor mails such as service state changes, ping failures, SSH login alerts, SMART warnings, filesystem alerts, SSL warnings, and power/throttling alerts generated by the Pi monitor script.

Recommended path shape:

- `YYYY-MM/YYYY-MM-DDTHH-MM-SS_check_status.txt`

Relevant code

- timer: `/etc/systemd/system/pi-monitor.timer`
- service: `/etc/systemd/system/pi-monitor.service`
- script: `/srv/monitoring/bin/monitor-checks.sh`

```ini
[Timer]
OnUnitActiveSec=1min
```

```sh
check_and_alert() {
  ...
  send_email "[Pi Monitor] $name $status" "$msg"
}
```
